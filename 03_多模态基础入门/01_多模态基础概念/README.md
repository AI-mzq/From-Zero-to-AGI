# 01_多模态基础概念

> 多模态不是“把图片、文字、声音都塞进模型”这么简单。它真正关心的是：不同形式的信息如何被表示、对齐、融合、推理，并最终服务于理解、生成和行动。

**By：猫先生 of 「魔方AI空间」**

## 本章导读

学习多模态，最容易一开始就被模型名带着跑：CLIP、BLIP、Flamingo、LLaVA、Qwen-VL、GPT-4V、Gemini、GPT-4o……

这些模型当然重要，但如果基础概念不清楚，很容易把多模态理解成“能传图片的 LLM”。这会漏掉几个关键问题：

- 什么是模态？
- 多模态和跨模态有什么区别？
- 多模态理解、生成、检索、交互分别在解决什么问题？
- VLM、MLLM、统一多模态模型是不是一回事？
- 为什么多模态模型更容易遇到幻觉、对齐和评测问题？

本章先搭概念地基。后面看具体模型时，就能知道每个模块在系统里负责什么，而不是只记住一个模型名字。

## 一句话理解多模态

多模态可以理解为：

> 让模型在文本、图像、视频、音频等不同信息形式之间建立联系，并基于这些信息完成理解、生成、检索、推理或行动。

这里有三个关键词。

| 关键词 | 含义 |
| --- | --- |
| 表示 | 把不同模态变成模型可计算的特征或 Token |
| 对齐 | 建立不同模态之间的语义对应关系 |
| 任务 | 面向问答、检索、生成、交互、工具调用等目标使用这些表示 |

如果只支持多种输入格式，但模型不能真正利用它们之间的关系，那还谈不上成熟的多模态能力。

## 什么是模态？

模态是信息的表达形式。

常见模态包括：

| 模态 | 典型数据 | 主要特点 |
| --- | --- | --- |
| 文本 | 句子、文档、代码、表格文字 | 离散、符号化、结构清晰 |
| 图像 | 照片、截图、海报、医学影像 | 二维空间结构强 |
| 视频 | 短视频、监控、直播、动作序列 | 同时包含空间和时间信息 |
| 音频 | 语音、音乐、环境声 | 连续信号，时间敏感 |
| 文档 | PDF、扫描件、表格、图表 | 同时包含文字、版面和视觉结构 |
| GUI | 软件界面、网页、移动端截图 | 视觉结构和可操作元素共存 |
| 3D / 传感器 | 点云、深度图、机器人状态 | 与空间、物理环境和行动相关 |

文本天然适合被拆成 Token。图像、视频、音频则需要先经过编码器，变成模型可处理的表示。

这也是多模态系统的第一步：

```text
非文本信号
  -> 模态编码器
  -> 特征 / Token
  -> 与语言模型或生成模型交互
```

## 多模态、跨模态和单模态

这几个词经常混用，但最好先区分清楚。

| 概念 | 简单解释 | 例子 |
| --- | --- | --- |
| 单模态 | 只处理一种模态 | 纯文本 LLM、图像分类模型、语音识别模型 |
| 跨模态 | 在两种或多种模态之间建立对应关系 | 图文检索、图片生成文字描述、文字生成图片 |
| 多模态 | 同时利用多种模态完成任务 | 图像问答、视频理解、语音对话、文档问答 |

跨模态更强调“模态之间的桥”，多模态更强调“多种信息一起完成任务”。

例如：

- 给一张图，找最匹配的文字描述，是跨模态检索。
- 给一张图和一个问题，让模型回答，是多模态理解。
- 给一句话生成一张图，是跨模态生成。
- 给一段视频、一段语音和用户问题，让模型完成任务，是更复杂的多模态交互。

## 多模态任务分类

多模态任务可以按目标分为几类。

| 类型 | 目标 | 常见任务 |
| --- | --- | --- |
| 理解 | 从非文本模态中提取语义 | VQA、OCR、图表理解、视频问答 |
| 检索 | 在不同模态之间寻找匹配项 | 图文检索、文档图片检索、视频片段检索 |
| 生成 | 从一种或多种模态生成另一种模态 | 文生图、图生文、文生视频、语音合成 |
| 推理 | 基于多模态信息做多步判断 | 视觉推理、图表推理、长视频事件推理 |
| 交互 | 在多轮上下文中理解和回应 | 多图对话、实时语音助手、GUI Agent |
| 行动 | 基于多模态输入调用工具或执行操作 | 屏幕操作、机器人控制、工作流自动化 |

这几个类型不是互斥的。真实系统往往会组合使用。

例如一个文档问答系统可能同时包含：

```text
PDF 解析
  -> OCR
  -> 图表理解
  -> 检索
  -> LLM 回答
  -> 引用定位
```

它不是单纯的“问答”，而是一个多模态系统。

## 多模态系统的基本结构

一个典型视觉语言模型可以简化成：

```text
图像 / 视频 / 音频
  -> 模态编码器
  -> 连接器
  -> LLM
  -> 文本回答 / 工具调用 / 多模态输出
```

其中每个模块都有明确分工。

| 模块 | 作用 | 常见形式 |
| --- | --- | --- |
| 模态编码器 | 把图像、视频或音频变成特征 | ViT、CLIP Vision Encoder、Audio Encoder |
| 连接器 | 把模态特征映射到 LLM 可接收空间 | Linear Projector、MLP、Q-Former、Cross-Attention |
| LLM 底座 | 负责语言理解、推理、指令遵循和输出 | LLaMA、Qwen、Mistral、DeepSeek 等 |
| 训练数据 | 提供图文对齐、指令遵循和领域知识 | 图文对、VQA、OCR、文档、视频文本数据 |
| 评测体系 | 检查模型是否真的理解输入 | VQA、OCR、MMBench、MMMU、幻觉评测 |

这也是为什么多模态模型不只是“视觉模型 + LLM”的简单加法。连接器、数据和评测同样决定能力边界。

## VLM、MLLM 和统一多模态模型

这些概念没有完全统一的行业定义，但可以按能力范围粗略区分。

| 名称 | 常见含义 | 典型能力 |
| --- | --- | --- |
| VLM | Vision-Language Model，视觉语言模型 | 图文检索、图像描述、视觉问答、OCR |
| MLLM | Multimodal Large Language Model，多模态大语言模型 | 以 LLM 为核心，接入图像、视频、音频等模态 |
| Unified Multimodal Model | 统一多模态模型 | 尝试用统一架构处理多种输入和输出模态 |

可以这样理解：

```text
VLM 更强调视觉与语言的连接
MLLM 更强调以 LLM 为中枢的多模态理解和交互
统一多模态模型更强调输入输出形态的统一
```

这三者不是严格割裂的。很多现代模型同时具备 VLM 和 MLLM 的特征。

## 多模态为什么难？

多模态的困难主要来自五个方面。

### 1. 表示差异

文本是离散符号，图像是二维空间信号，视频还有时间维度，音频是连续波形。

不同模态的信息结构差异很大，不能直接放进同一个模型里处理。

### 2. 对齐困难

一句话里的“红色杯子”，需要和图像中的具体区域对应起来。视频里的“他转身离开”，需要和一段连续动作对应起来。

模型不仅要识别内容，还要知道语言和视觉证据之间的对应关系。

### 3. 信息密度不同

一张图可能包含大量细节；一段视频可能包含数百帧；一份 PDF 可能包含文字、表格、图像和版面。

如果全部转成 Token，会消耗大量上下文和计算资源。

### 4. 数据质量不稳定

图文对数据可能噪声很大。网页图片和标题不一定严格对应；视频字幕不一定描述画面；OCR 数据也可能有识别错误。

多模态模型对数据质量非常敏感。

### 5. 评测更复杂

文本任务可以用标准答案评估一部分能力。多模态任务还要判断模型是否真的基于图像、视频或音频作答。

这也是多模态幻觉更难处理的原因。

## 多模态幻觉

多模态幻觉指模型输出了输入中并不存在、无法支持或明显错误的内容。

常见情况包括：

| 类型 | 例子 |
| --- | --- |
| 对象幻觉 | 图中没有狗，模型却说有狗 |
| 属性幻觉 | 图中衣服是蓝色，模型说成红色 |
| 关系幻觉 | 两个人没有互动，模型说他们正在握手 |
| OCR 幻觉 | 文档中没有某句话，模型却编出引用 |
| 视频事件幻觉 | 视频里没有摔倒，模型说有人摔倒 |

多模态幻觉比纯文本幻觉更隐蔽，因为输出通常听起来很自然。评测时必须回到输入证据本身。

## 多模态能力的三个层级

可以用三个层级来判断一个模型的多模态能力。

| 层级 | 能力 | 判断方式 |
| --- | --- | --- |
| 看见 | 能识别输入中显性的对象、文字、场景 | OCR、对象识别、简单 VQA |
| 看懂 | 能理解关系、结构、意图和上下文 | 图表问答、视觉推理、多图对比 |
| 会用 | 能把多模态信息接入任务和行动 | 文档 RAG、GUI Agent、工具调用 |

很多模型能做到“看见”，但不一定稳定做到“看懂”；能回答图像问题，也不代表能在业务系统中可靠“会用”。

这个区分很重要。

## 多模态与 LLM 的关系

现代 MLLM 通常以 LLM 为核心，但多模态能力并不完全来自 LLM。

可以把分工理解为：

| 部分 | 主要职责 |
| --- | --- |
| 模态编码器 | 提取图像、视频、音频等非文本特征 |
| 连接器 | 把模态特征转成 LLM 可理解的表示 |
| LLM | 负责语言推理、指令遵循、上下文组织和输出 |
| 数据与训练 | 决定模型是否能把视觉证据和语言任务对齐 |

因此，一个强 LLM 不一定自动得到强多模态能力；一个强视觉编码器也不一定自动得到好对话体验。

真正的多模态能力来自模块组合、训练数据和评测闭环。

## 多模态与传统 CV / NLP 的关系

多模态大模型不是传统 CV / NLP 的替代品，而是在它们基础上的系统整合。

| 传统方向 | 典型问题 | 多模态大模型中的变化 |
| --- | --- | --- |
| CV | 图像分类、检测、分割、OCR | 更强调开放词表、指令遵循和语言交互 |
| NLP | 问答、摘要、翻译、推理 | 输入不再只是文本，还包含图像、视频、音频 |
| Speech | ASR、TTS、语音理解 | 更强调实时交互和语音-语言-行动闭环 |
| IR | 文本检索、图像检索 | 扩展到多模态 RAG 和证据引用 |

多模态大模型的价值不在于抛弃旧任务，而是把这些任务放进统一的交互和推理框架里。

## 常见误区

### 1. 能上传图片就是多模态能力强

图片输入只是入口。关键要看模型是否能正确理解细节、结构、文字和上下文。

### 2. VLM 等于 MLLM

VLM 更偏视觉和语言连接，MLLM 更强调以 LLM 为核心的多模态理解、推理和交互。两者有重叠，但不完全相同。

### 3. 图文对齐好就能做复杂视觉推理

图文对齐是基础，但复杂推理还需要指令数据、推理能力、训练策略和评测约束。

### 4. 多模态只是图像加文本

图像和文本只是最常见起点。视频、音频、文档、GUI、3D 和传感器都会进入多模态系统。

### 5. 多模态模型的回答都可信

多模态模型同样会幻觉。尤其在 OCR、文档、医疗、图表和视频事件中，必须检查证据。

## 前瞻性判断

多模态基础概念会随着模型演进而扩展，但有几条趋势值得长期关注。

### 1. 从图文对齐走向多模态统一表示

CLIP 和 ALIGN 这类方法奠定了图文对齐的基础。后续模型会继续尝试把图像、视频、音频、文本、3D 等模态放进更统一的表示空间。

ImageBind 就是这一方向的代表之一，它尝试把图像、文本、音频、深度、热成像、IMU 等多种模态绑定到同一嵌入空间。

### 2. 从单图理解走向长上下文多模态

真实任务往往不是看一张图，而是读一份 PDF、理解一段视频、比较多张图片，或者在多轮对话中持续引用视觉信息。

长上下文、多图、多页、多帧会成为基础能力。

### 3. 从回答问题走向执行任务

未来多模态模型不会只回答“图里有什么”，而会越来越多地参与任务执行：

- 读文档并填写表格
- 看屏幕并操作软件
- 看视频并提取事件
- 听语音并调用工具
- 根据现场信息辅助决策

### 4. 从能力展示走向可靠性工程

多模态模型进入真实业务后，评测、安全、引用、可解释性和错误恢复会变得更重要。

长期看，多模态系统的竞争不只在模型参数，也在数据治理、评测体系和工程闭环。

## 本章小结

多模态基础可以压缩成一条主线：

```text
不同模态
  -> 编码成表示
  -> 建立语义对齐
  -> 接入语言推理和生成
  -> 面向任务进行评测和应用
```

学习多模态，先不要急着追模型名。把“模态、表示、对齐、任务、评测、应用”这几个基础概念理顺，后面再看 CLIP、BLIP-2、Flamingo、LLaVA、Qwen-VL、GPT-4V 等模型，会清楚很多。

## 推荐阅读与引用来源

- [Learning Transferable Visual Models From Natural Language Supervision](https://arxiv.org/abs/2103.00020)：CLIP，图文对比学习和开放词表视觉能力的重要基础。
- [Scaling Up Visual and Vision-Language Representation Learning With Noisy Text Supervision](https://arxiv.org/abs/2102.05918)：ALIGN，使用大规模噪声图文对训练视觉语言表示。
- [BLIP-2: Bootstrapping Language-Image Pre-training with Frozen Image Encoders and Large Language Models](https://arxiv.org/abs/2301.12597)：提出 Q-Former，把冻结视觉编码器和冻结 LLM 连接起来。
- [Flamingo: a Visual Language Model for Few-Shot Learning](https://arxiv.org/abs/2204.14198)：早期强代表的少样本视觉语言模型。
- [Visual Instruction Tuning](https://arxiv.org/abs/2304.08485)：LLaVA，视觉指令微调路线的代表工作。
- [ImageBind: One Embedding Space To Bind Them All](https://arxiv.org/abs/2305.05665)：探索多种模态共享嵌入空间。
- [GPT-4 Technical Report](https://arxiv.org/abs/2303.08774)：包含多模态能力、安全和评测讨论。
- [Gemini: A Family of Highly Capable Multimodal Models](https://arxiv.org/abs/2312.11805)：Gemini 技术报告，展示原生多模态模型路线。
