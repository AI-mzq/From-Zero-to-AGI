# 05_视觉语言模型 VLM

> 视觉语言模型（Vision-Language Model, VLM）让模型能够在图像与文本之间建立语义连接，并进一步完成图像描述、视觉问答、图文检索、文档理解、多轮图文对话和视觉推理。

## 一句话理解 VLM

VLM 可以理解为：

> 让模型把“看见的视觉内容”和“读懂的自然语言”放到同一个任务空间里，并能根据文本指令使用视觉信息。

早期 VLM 更偏向图文匹配和检索：

```text
图片 <-> 文本描述
```

现代 VLM 更偏向对话、推理和任务执行：

```text
图像 / 截图 / 文档 / 图表
  + 用户指令
  -> 多模态理解
  -> 自然语言回答 / 结构化输出 / 工具调用
```

所以学习 VLM 时，不要只把它理解成“看图说话”。它真正重要的地方，是把视觉信息纳入语言模型的上下文和推理流程。

## VLM、MLLM 和 LVLM 的关系

这几个词经常混用，但侧重点略有不同。

| 名称 | 全称 | 侧重点 |
| --- | --- | --- |
| VLM | Vision-Language Model | 图像和文本之间的联合建模，范围最广 |
| LVLM | Large Vision-Language Model | 大规模视觉语言模型，通常强调大模型规模 |
| MLLM | Multimodal Large Language Model | 以 LLM 为中枢，接入图像、视频、音频等多模态输入 |

在很多语境下，现代 VLM 已经接近 MLLM 的一部分：它通常由视觉编码器、连接器和 LLM 底座组成，能够进行图文对话和视觉推理。

但严格说，MLLM 的范围更大。它不仅包括图像文本，还可能包括视频、音频、语音、3D、传感器、GUI 操作和工具调用。

本章重点讨论图像-文本方向的视觉语言模型。

## 一个典型 VLM 的结构

现代 VLM 的基本结构可以写成：

```text
图像
  -> 视觉编码器
  -> 视觉特征
  -> 连接器 / Resampler / Cross-Attention
  -> LLM 上下文
  -> 文本回答
```

如果加入用户问题，就是：

```text
图像 + 文本指令
  -> 多模态上下文
  -> LLM 解码
  -> 答案 / 描述 / 推理过程 / JSON / 工具调用
```

核心不是简单“把图像变成文字”，而是让模型在生成每个回答 Token 时，能够参考视觉信息和文本上下文。

## VLM 的三条主线

VLM 的发展可以粗略看成三条主线交织：

| 主线 | 代表模型 | 核心问题 |
| --- | --- | --- |
| 图文对齐 | CLIP、ALIGN、SigLIP | 图像和文本如何进入同一个语义空间？ |
| 图文理解与生成 | BLIP、BLIP-2、PaLI | 模型如何既能理解图像，又能生成文本？ |
| 多模态助手 | Flamingo、LLaVA、InstructBLIP、Qwen-VL、InternVL | 模型如何遵循人类指令，完成开放式图文任务？ |

这三条线不是互斥关系。现代 VLM 往往同时吸收了图文对齐、视觉指令微调、强 LLM 底座和高分辨率视觉建模。

## CLIP：图文对比学习的起点

CLIP 的核心思想很朴素：把图像和文本分别编码成向量，然后让匹配的图文对更接近，不匹配的图文对更远。

```text
image_encoder(image) -> image_embedding
text_encoder(text)   -> text_embedding

匹配图文：相似度更高
不匹配图文：相似度更低
```

CLIP 的价值在于，它让视觉模型具备了开放词表语义能力。

传统分类模型通常只能识别训练好的类别。CLIP 则可以用自然语言描述作为类别提示，例如：

```text
"a photo of a cat"
"a photo of a dog"
"a photo of a street sign"
```

然后比较图像和这些文本描述的相似度。

这带来了几个重要影响：

- 图文检索能力显著增强。
- 视觉表示开始具备开放语义。
- 后续很多 VLM 直接使用 CLIP 视觉编码器作为“眼睛”。
- 视觉特征更容易通过连接器接入 LLM。

CLIP 不是现代多模态助手本身，但它是非常关键的地基之一。

## ALIGN：更大规模的弱监督图文对齐

ALIGN 和 CLIP 类似，也采用图文对比学习，但更强调使用大规模噪声图文对进行弱监督训练。

它说明了一个方向：多模态对齐不一定依赖精标数据，大规模网页图文对也能带来强泛化能力。

这对后续 VLM 很重要，因为真实世界的图文数据往往噪声很大：

- 图片标题不精确。
- 网页 alt 文本质量参差不齐。
- 图文可能只是弱相关。
- 数据规模巨大，人工清洗成本高。

VLM 的一个长期主题，就是如何在大规模弱监督数据、合成数据和高质量指令数据之间取得平衡。

## BLIP：从图文对齐走向理解与生成

CLIP 主要解决图文匹配问题，但它不是一个强文本生成模型。

BLIP 进一步把图文理解和文本生成结合起来，面向图像描述、视觉问答、图文检索等任务构建统一框架。

可以把 BLIP 理解为从“图文相似度”向“图文任务模型”的过渡：

```text
CLIP 类路线：
  更擅长图文匹配和检索

BLIP 类路线：
  同时支持理解、检索和生成
```

BLIP 还强调数据质量问题，通过生成和过滤更高质量的图文样本，改善视觉语言预训练效果。

这点非常重要。多模态训练不是只要数据越多越好，图文对应是否准确、描述是否丰富、是否覆盖细粒度内容，都会影响模型能力。

## BLIP-2：用 Q-Former 连接视觉模型和语言模型

BLIP-2 的关键贡献，是提出用 Q-Former 连接冻结的视觉编码器和冻结的语言模型。

它的核心思路是：

```text
冻结视觉编码器
  -> 输出大量视觉特征
  -> Q-Former 用少量 Query 提取视觉信息
  -> 接入冻结 LLM
  -> 生成文本
```

这种设计的意义在于：

- 不必从零训练完整多模态大模型。
- 可以复用已有强视觉编码器和强语言模型。
- Q-Former 起到视觉信息压缩和对齐作用。
- 训练成本相对更可控。

BLIP-2 代表了一种很有影响力的思路：与其把所有模块一起暴力训练，不如让连接器承担跨模态适配任务。

## Flamingo：把视觉上下文接入语言模型

Flamingo 代表了另一条重要路线：在强语言模型中加入视觉信息读取机制，使模型能够处理交错的图文上下文，并具备多模态 few-shot 能力。

一个典型输入可以是：

```text
图像 1 + 问题 1 + 答案 1
图像 2 + 问题 2 + 答案 2
图像 3 + 新问题
```

模型需要从前面的图文示例中学习任务格式，再回答新图像的问题。

Flamingo 中的 Perceiver Resampler 用于压缩视觉特征，Cross-Attention 层让语言模型在生成时读取视觉信息。

这条路线的启发是：

- VLM 不只是单图单问答。
- 图文可以交错出现在上下文中。
- 多模态 few-shot 是通向通用视觉助手的重要能力。
- 视觉信息的接入方式会影响模型能否稳定处理多轮、多图和长上下文。

## LLaVA：视觉指令微调成为主线

LLaVA 的影响很大，因为它把 VLM 推向了“多模态聊天助手”的形态。

它的基本结构很清晰：

```text
CLIP 视觉编码器
  -> MLP Projector
  -> LLM
  -> 视觉指令微调
```

LLaVA 的关键不只是结构，而是训练范式：用 GPT-4 等模型生成视觉指令数据，让模型学习如何根据图片回答人类问题。

这一步让 VLM 从“会匹配图文”进一步走向：

- 会听指令。
- 会解释图片。
- 会进行多轮图文对话。
- 会按用户要求输出摘要、判断、比较和结构化信息。

这也是现代开源 VLM 常见训练路线的起点之一：

```text
预训练对齐
  -> 视觉指令微调
  -> 多任务混合训练
  -> 偏好对齐 / 安全对齐
```

## InstructBLIP：把指令学习引入 BLIP-2 路线

InstructBLIP 可以看作 BLIP-2 思路和指令微调思路的结合。

它继续使用 Q-Former 连接视觉编码器和语言模型，同时引入多任务视觉指令数据，让模型更好地适配开放式任务。

它说明了一个趋势：连接器解决“图像怎么接入 LLM”，指令微调解决“模型怎么按人类意图使用图像”。

这两件事缺一不可。

## 开源 VLM 的共同模式

后来大量开源 VLM 在结构上有不同选择，但基本模式高度相似。

```text
强视觉编码器
  + 连接器
  + 强 LLM 底座
  + 多模态预训练数据
  + 视觉指令微调数据
  + 多任务评测
```

代表性模型包括 Qwen-VL、InternVL、MiniGPT-4、CogVLM、DeepSeek-VL、MiniCPM-V 等。

它们的差异通常体现在：

| 差异点 | 典型问题 |
| --- | --- |
| 视觉编码器 | 使用 CLIP、EVA-CLIP、SigLIP 还是自研视觉骨干？ |
| 分辨率策略 | 固定分辨率、动态分辨率还是切图策略？ |
| 连接器 | MLP Projector、Q-Former、Cross-Attention 还是混合设计？ |
| LLM 底座 | 使用哪一类语言模型，是否具备强推理和长上下文？ |
| 训练数据 | 图文对、OCR、文档、图表、数学、代码、GUI 是否覆盖？ |
| 对齐方式 | 是否做视觉指令微调、安全对齐、偏好优化？ |
| 部署目标 | 偏研究评测、端侧部署、企业文档还是 Agent 场景？ |

所以比较 VLM 时，不能只问“谁分数更高”，还要问它在哪些数据、分辨率、任务和部署条件下更强。

## 闭源视觉模型怎么看？

GPT-4V、Gemini、Claude Vision 等闭源模型推动了多模态产品形态的发展。它们通常在通用视觉问答、文档理解、截图理解、多图对比和复杂指令遵循上表现较强。

但对学习和研究来说，闭源模型也有明显限制：

- 训练数据和架构细节不可见。
- 能力边界依赖系统策略和产品版本。
- 复现实验困难。
- Benchmark 分数不一定能解释能力来源。

因此本项目更建议把闭源模型当作能力参照，而不是学习结构的唯一入口。要理解 VLM 的技术脉络，仍然要回到公开论文、开源模型和可复现实验。

## VLM 的典型任务

### 1. 图像描述

Image Captioning 要求模型把图像内容转成自然语言描述。

它看似简单，实际考察的是：

- 是否能识别主体。
- 是否能保留关键属性。
- 是否能描述关系和场景。
- 是否避免编造不存在的细节。
- 是否能根据用户要求控制描述粒度。

例如：

```text
用户：请用一句话描述这张图。
模型：一名骑行者正沿着城市道路经过一排停放的汽车。
```

更强的模型还应支持风格、长度、结构和用途控制。

### 2. VQA

Visual Question Answering 要求模型基于图像回答问题。

问题可能很简单：

```text
图中有几个人？
```

也可能需要推理：

```text
根据图中人物的动作，接下来最可能发生什么？
```

VQA 不只是识别物体，还涉及指令理解、视觉定位、关系判断和语言生成。

### 3. OCR 与文档理解

OCR / Document Understanding 是 VLM 非常重要的应用方向。

它要求模型不仅看到图片，还要读懂图片里的文字、表格、版面和结构。

典型任务包括：

- 识别截图中的文字。
- 阅读 PDF 页面。
- 从发票、合同、表格中抽取字段。
- 根据文档内容回答问题。
- 保持表格、编号和引用关系。

这类任务对高分辨率、局部细节和版面结构非常敏感。

### 4. 图表理解

图表理解要求模型读懂柱状图、折线图、饼图、散点图、仪表盘等可视化内容。

它不仅要识别视觉元素，还要理解：

- 坐标轴
- 图例
- 数值
- 趋势
- 异常点
- 对比关系

这类任务强依赖 LLM 的数值推理和解释能力。视觉看清楚只是第一步，答得准还需要语言模型能进行合理计算和归纳。

### 5. 图文检索

图文检索是早期 VLM 的经典任务。

常见形式包括：

- 给一段文本，找最匹配的图片。
- 给一张图片，找最匹配的描述。
- 在图片库中检索相似语义内容。

CLIP、ALIGN 这类模型在这一方向非常有代表性。

### 6. 多图对比

多图任务要求模型同时处理多张图，并进行比较、归纳或选择。

例如：

```text
比较这两张产品图，指出主要差异。
```

难点在于模型要稳定区分不同图像的内容，避免把图 A 的细节混到图 B 上。

### 7. 视觉推理

视觉推理要求模型基于视觉信息做多步判断。

例如：

- 根据图片判断物体之间的空间关系。
- 根据流程图推断下一步。
- 根据图表数据解释业务趋势。
- 根据截图判断用户应该点击哪里。

这类任务往往同时考察视觉细节、语言理解和推理能力。

## VLM 能力的关键决定因素

一个 VLM 强不强，通常由多个因素共同决定。

| 因素 | 影响 |
| --- | --- |
| 视觉编码器 | 决定图像特征质量、分辨率适配和细节保留 |
| 连接器 | 决定视觉信息如何进入 LLM，以及压缩是否损失关键信息 |
| LLM 底座 | 决定语言表达、推理、指令遵循和长上下文能力 |
| 训练数据 | 决定任务覆盖、知识范围、OCR、文档和图表能力 |
| 指令微调 | 决定模型能否按人类意图使用视觉信息 |
| 分辨率策略 | 决定小字、图表、GUI、复杂场景是否看得清 |
| 评测覆盖 | 决定能力是否被真实暴露，而不是只在少数榜单上好看 |

这也是为什么有些模型参数不大，但在 OCR 或文档任务上很强；有些模型语言底座很强，但视觉细节识别并不一定稳定。

## VLM 的训练流程

不同模型细节不同，但常见训练流程可以概括为：

```text
阶段 1：图文对齐预训练
  让视觉特征和文本语义初步对齐

阶段 2：连接器 / 多模态预训练
  让视觉 Token 能进入 LLM 并参与生成

阶段 3：视觉指令微调
  让模型学会按用户问题使用图像

阶段 4：任务增强训练
  加入 OCR、文档、图表、数学、GUI、多图等专项数据

阶段 5：偏好对齐与安全对齐
  改善回答质量、拒答边界和不确定性表达
```

其中，阶段 3 是从“模型能看图”走向“模型能作为助手用图”的关键一步。

## VLM 的评测维度

评测 VLM 不应该只看一个综合分。

更合理的评测维度包括：

| 维度 | 典型问题 |
| --- | --- |
| 感知能力 | 能否识别物体、属性、数量、颜色、位置？ |
| 细粒度理解 | 能否读小字、看图表、理解复杂版面？ |
| 指令遵循 | 是否按用户要求回答，而不是泛泛描述？ |
| 推理能力 | 能否进行多步比较、归纳、计算和判断？ |
| 多图能力 | 能否区分不同图片并做跨图比较？ |
| 幻觉控制 | 是否会编造图中不存在的内容？ |
| 不确定性表达 | 看不清时是否会说明限制？ |
| 稳定性 | 换一种问法、裁剪或分辨率后是否仍然可靠？ |
| 部署成本 | Token 数、延迟、显存、吞吐是否可接受？ |

一个严肃的 VLM 评估，应同时包含通用 Benchmark、人工样例、业务样例和失败案例分析。

## VLM 的常见失败模式

### 1. 视觉幻觉

模型可能描述图中不存在的物体、文字或关系。

这通常来自：

- 视觉信息不足。
- 语言先验过强。
- 训练数据中常见模式误导。
- 模型没有学会表达不确定。

### 2. OCR 错误

小字、低清晰度、复杂背景、手写体、倾斜文本都会导致 OCR 失败。

即使模型能大致看懂场景，也不代表它能准确读出文字。

### 3. 数量和位置错误

VLM 经常在计数、左右关系、前后遮挡、细粒度位置上出错。

这类错误在看似简单的图片问答中也很常见。

### 4. 多图混淆

当输入多张图时，模型可能把不同图片的信息混在一起。

这要求系统在提示模板、图像标记和训练数据中明确区分图像编号。

### 5. 图表推理不稳

图表任务往往要同时读视觉、读数值、理解图例和做计算。任何一步出错，最终答案都会偏离。

### 6. 对提示过度迎合

如果用户在问题里暗示了错误结论，模型可能顺着用户说，而不是回到视觉证据。

可靠 VLM 应该能在视觉证据不足或与问题前提冲突时明确指出。

## 如何阅读一篇 VLM 论文？

读 VLM 论文时，可以按下面几个问题拆解：

1. **视觉编码器是什么？** 使用 CLIP、SigLIP、EVA-CLIP、自研 ViT，还是多尺度视觉骨干？
2. **连接器怎么设计？** 是 MLP Projector、Q-Former、Perceiver Resampler、Cross-Attention，还是其他结构？
3. **LLM 底座是什么？** 底座大小、上下文长度、语言和推理能力如何？
4. **训练分几阶段？** 哪些阶段冻结，哪些阶段训练？
5. **数据来自哪里？** 图文对、指令数据、OCR、文档、图表、多图、合成数据是否覆盖？
6. **评测覆盖什么任务？** 是否只测通用 VQA，还是包含真实复杂场景？
7. **失败案例有没有分析？** 是否讨论幻觉、OCR、计数、位置、多图混淆？
8. **部署成本如何？** 分辨率、视觉 Token 数、推理延迟和显存是否可接受？

按这套问题读，基本能看清一篇 VLM 论文的真实贡献和边界。

## 代表路线对比

| 路线 | 代表模型 | 核心贡献 | 局限 |
| --- | --- | --- | --- |
| 图文对比学习 | CLIP、ALIGN | 建立开放语义图文空间 | 不擅长复杂生成和多轮对话 |
| 统一理解生成 | BLIP | 结合图文理解、检索和生成 | 对大语言模型式对话支持有限 |
| 连接器适配 | BLIP-2 | Q-Former 连接冻结视觉模型和 LLM | Query 压缩可能损失细节 |
| 视觉上下文 LLM | Flamingo | 支持交错图文上下文和 few-shot | 结构与训练成本较高 |
| 视觉指令微调 | LLaVA、InstructBLIP | 让模型学会按人类指令看图回答 | 数据质量和覆盖面决定上限 |
| 开源通用 VLM | Qwen-VL、InternVL、MiniCPM-V 等 | 覆盖 OCR、文档、图表、多图等真实任务 | 能力差异强依赖训练细节和版本 |

## VLM 与多模态 Agent 的关系

VLM 是多模态 Agent 的感知入口之一。

在 GUI Agent、文档 Agent、网页 Agent、机器人和自动化工作流中，模型经常需要先看懂视觉环境，再决定下一步动作。

例如：

```text
截图
  -> VLM 理解当前界面
  -> LLM 分析用户目标
  -> 工具调用 / 点击 / 输入 / 检索
  -> 观察新状态
  -> 继续决策
```

这时 VLM 不能只会描述图片，还要能支持：

- 定位控件。
- 读取界面文字。
- 理解状态变化。
- 关联历史操作。
- 输出可执行动作。

所以 VLM 和 Agent 的结合，会把“视觉理解”推进到“视觉环境中的行动”。

## 前瞻性判断

VLM 后续仍会沿着几个方向继续演进。

### 1. 从单图问答走向复杂视觉任务

简单图像描述和 VQA 已经不是终点。文档、图表、GUI、多图、长视频和真实业务场景会更能拉开差距。

### 2. 从固定分辨率走向动态视觉预算

模型需要根据任务动态决定看哪里、看多细、保留多少视觉 Token，而不是所有图片都用同一种分辨率处理。

### 3. 从回答正确走向证据可靠

未来 VLM 不只要给答案，还要能指出答案来自图像中的哪一部分，尤其在文档、医疗、法律、金融等高风险场景中。

### 4. 从图文模型走向全模态模型

图像只是起点。视频、语音、音频、屏幕、动作、工具反馈会逐步进入同一上下文。

### 5. 从模型能力走向系统能力

真实应用里，VLM 往往要和 OCR、RAG、检测、分割、代码执行、浏览器和业务数据库协同。模型本身重要，系统设计同样重要。

## 常见误区

### 1. VLM 不是把图像转成一段 caption 再喂给 LLM

有些系统确实会用 caption 作为中间层，但现代 VLM 通常直接把视觉 Token 接入 LLM 上下文。

### 2. 强 LLM 不等于强 VLM

LLM 底座强，只说明语言和推理基础好。视觉编码器、连接器、分辨率策略和多模态数据同样决定最终能力。

### 3. 看得见不等于看得准

模型可能能描述整体场景，但在小字、数量、位置、图表数值上出错。

### 4. Benchmark 高不等于业务可用

真实业务常常有低清晰度截图、复杂表格、多页 PDF、非标准图表和长上下文。只看公开榜单容易高估可用性。

### 5. VLM 不会天然知道自己看不清

很多模型会在视觉证据不足时继续猜。可靠系统需要训练、提示和评测共同约束不确定性表达。

## 本章小结

VLM 的核心可以概括为：

```text
视觉编码器负责提取图像特征
连接器负责跨模态对齐
LLM 负责指令理解、推理和生成
训练数据决定模型会如何使用视觉信息
评测和系统设计决定模型能否可靠落地
```

CLIP 让图像和文本进入同一个语义空间，BLIP / BLIP-2 推动图文理解与生成，Flamingo 强调交错图文上下文和 few-shot，LLaVA 把视觉指令微调变成开源 VLM 的主线。

今天的 VLM 已经不只是“看图说话”，而是在逐步成为多模态 Agent、文档智能、视频理解和真实世界自动化系统的感知入口。
