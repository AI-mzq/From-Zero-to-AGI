# 06_多模态生成模型

> 多模态生成模型关注文本、图像、视频、音频等模态之间的生成、转换、编辑和控制。它不是某一种模型结构，而是一组围绕“如何把意图变成内容”的长期技术体系。

**By：猫先生 of 「魔方AI空间」**

## 本章导读

前面一章讲的是 [视觉语言模型 VLM](../05_视觉语言模型VLM/README.md)：模型如何看懂图像，并基于图像回答问题。

这一章转向另一个方向：模型如何生成内容。

```text
理解模型：
  图像 / 视频 / 文档 -> 文本回答

生成模型：
  文本 / 图像 / 草图 / 音频 / 条件 -> 图像 / 视频 / 音频 / 3D / 多模态内容
```

多模态生成模型是 AIGC 的核心底座。文生图、图生图、图像编辑、文生视频、视频编辑、语音生成、音乐生成、3D 生成，都属于这个大方向。

本章重点回答：

- 多模态生成模型到底在生成什么？
- 扩散模型、自回归模型、掩码生成模型、流匹配模型分别解决什么问题？
- 图像 Tokenizer、VAE、Latent Space 为什么重要？
- 文本、图像、草图、深度图、姿态、边缘图如何控制生成？
- 图像生成、视频生成、音频生成和统一多模态生成的共性是什么？
- 为什么“可控性、一致性、可编辑性、可评测性”比单纯清晰度更长期？

## 一句话理解多模态生成

多模态生成可以理解为：

> 把一种或多种模态中的意图、条件和上下文，转换成另一种模态中的内容。

例如：

| 输入条件 | 输出内容 | 典型任务 |
| --- | --- | --- |
| 文本 | 图像 | 文生图 |
| 文本 + 图像 | 修改后的图像 | 图像编辑 |
| 文本 + 草图 | 图像 | 草图控制生成 |
| 文本 + 姿态 | 人物图像 | 姿态可控生成 |
| 文本 + 首帧 | 视频 | 图生视频 / 首帧生成视频 |
| 文本 | 音频 / 语音 / 音乐 | 文生音频 |
| 图像 + 文本 + 音频 | 统一多模态内容 | 多模态创作系统 |

无论模型名字怎么变，核心问题长期不会变：

```text
如何表示内容？
如何表达条件？
如何生成高质量样本？
如何让生成结果可控、可编辑、可复现？
如何评估它是不是忠实、安全、有用？
```

## 生成模型和理解模型的区别

VLM 更偏“理解”，多模态生成模型更偏“创造”。

| 维度 | 理解模型 | 生成模型 |
| --- | --- | --- |
| 输入 | 图像、视频、文本等 | 文本、图像、控制条件、历史上下文 |
| 输出 | 文本回答、标签、结构化信息 | 图像、视频、音频、3D、编辑结果 |
| 关键能力 | 感知、理解、推理、问答 | 采样、合成、编辑、控制、一致性 |
| 主要风险 | 看错、幻觉、漏读 | 失真、跑题、不一致、不可控 |
| 典型评估 | VQA、OCR、图表理解、幻觉 | 质量、对齐、真实感、多样性、可控性 |

当然，两者正在融合。

一个强多模态系统往往需要先理解，再生成：

```text
用户给一张图
  -> 模型理解内容和编辑意图
  -> 生成修改后的图像或视频
```

未来的多模态模型不会只回答问题，也不会只生成图片，而是能在理解、编辑、生成和行动之间连续切换。

## 多模态生成的长期主线

多模态生成模型可以沿着四条主线理解。

| 主线 | 核心问题 | 代表方向 |
| --- | --- | --- |
| 表示 | 内容如何变成模型可处理的表示？ | VAE、VQ-VAE、视觉 Tokenizer、Latent Space |
| 生成 | 模型如何从噪声或 Token 中生成内容？ | 扩散、自回归、Mask Modeling、Flow Matching |
| 条件 | 用户意图如何控制生成过程？ | Prompt、图像条件、ControlNet、参考图、布局 |
| 系统 | 如何把生成模型变成稳定产品？ | 编辑工作流、评测、安全、版权、推理优化 |

这四条线比具体模型名字更重要。模型会换，问题不会消失。

## 生成什么：像素、Token 还是潜变量？

生成模型首先要回答：输出内容用什么形式表示？

### 1. 直接生成像素

最直观的方式是直接生成像素。

```text
模型输出：
  RGB 像素矩阵
```

这种方式概念简单，但高分辨率图像的像素空间非常大，直接建模成本高。

早期很多生成模型在低分辨率图像上工作，到了高分辨率内容生成时，需要更高效的表示方式。

### 2. 生成离散视觉 Token

另一种方式是先用图像 Tokenizer 把图像压缩成离散 Token，再像语言模型一样生成这些 Token。

```text
图像
  -> 图像 Tokenizer
  -> 离散视觉 Token
  -> 自回归 / 掩码生成
  -> Token 序列
  -> 解码为图像
```

VQ-VAE、VQGAN、DALL-E、Parti、Muse 等路线都和离散视觉表示有关。

这种方式的优势是可以借鉴语言模型的序列建模能力；难点是视觉 Tokenizer 的质量会限制最终生成质量。

### 3. 生成连续潜变量

Latent Diffusion 的思路是：不在原始像素空间做扩散，而是在压缩后的连续潜空间中生成。

```text
图像
  -> VAE Encoder
  -> Latent
  -> Diffusion / Flow
  -> Generated Latent
  -> VAE Decoder
  -> 图像
```

这条路线极大影响了后来的文生图系统，因为它在质量、效率和可扩展性之间取得了很好的平衡。

理解“像素空间、离散 Token、连续 Latent”这三种表示，是理解多模态生成的入口。

## 扩散模型

扩散模型是近几年图像和视频生成中最重要的范式之一。

它的基本思路可以简化为：

```text
训练时：
  干净图像 -> 逐步加噪 -> 学习如何去噪

生成时：
  随机噪声 -> 逐步去噪 -> 生成图像
```

扩散模型的优势：

- 生成质量高。
- 训练相对稳定。
- 可以自然接入文本、图像、边缘、深度、姿态等条件。
- 适合图像编辑、局部重绘、超分辨率、视频生成等任务。

局限也很明显：

- 采样通常需要多步迭代。
- 推理速度和成本需要优化。
- 文本对齐、结构控制、长程一致性并不天然解决。
- 视频生成中时序一致性更难。

扩散模型不是“魔法图像引擎”，本质上仍然是在学习一个从噪声到数据分布的反向过程。

## Latent Diffusion：在潜空间中生成

Latent Diffusion 的关键思想是把扩散过程放到潜空间中，而不是原始像素空间。

一个简化流程是：

```text
图像
  -> VAE 压缩到 Latent
  -> 在 Latent 上进行扩散训练
  -> 生成 Latent
  -> VAE 解码回图像
```

它解决了一个实际问题：高分辨率像素空间太大，直接扩散成本很高。

潜空间生成的长期价值在于：

- 降低训练和推理成本。
- 保留足够视觉语义。
- 方便接入文本条件。
- 方便做编辑、重绘和控制。

Stable Diffusion 是 Latent Diffusion 影响力最大的落地之一，但这里更重要的是背后的思想：先学习一个高质量压缩空间，再在这个空间里生成。

## 自回归多模态生成

自回归生成是语言模型最熟悉的范式：一个 Token 接一个 Token 地生成。

如果图像、音频或视频也能被离散化为 Token，那么模型也可以用类似语言模型的方式生成多模态内容。

```text
文本 Token + 图像 Token
  -> Transformer
  -> 下一个 Token
  -> 继续生成
```

自回归路线的优势：

- 与 LLM 架构天然兼容。
- 适合统一建模文本、图像、音频等离散序列。
- 可以处理复杂上下文和多轮交互。
- 对“统一多模态模型”很有吸引力。

局限：

- 高质量视觉 Tokenizer 很难。
- 图像 Token 序列可能很长。
- 局部细节和全局结构都要靠序列生成维持。
- 采样速度和错误累积需要控制。

自回归生成的长期问题不是“能不能生成图像”，而是：能否用统一序列模型同时做到高保真视觉质量、稳定语言能力和可控生成。

## Mask Modeling 与并行生成

掩码生成模型不一定按从左到右的顺序生成，而是先遮住一部分 Token，再逐步预测缺失部分。

```text
部分可见 Token + Mask Token
  -> 预测被遮住的视觉 Token
  -> 多轮 refine
  -> 完整图像
```

这类方法的吸引力在于：

- 生成可以更并行。
- 适合图像这种二维结构。
- 可以用于图像补全、编辑和条件生成。
- 在离散视觉 Token 上较自然。

MaskGIT、Muse 等工作都体现了这条思路。

长期来看，掩码建模代表的是一种思想：生成不一定必须严格按单向顺序，也可以通过迭代补全来逼近目标内容。

## Flow Matching 与 Rectified Flow

除了扩散和自回归，流匹配、Rectified Flow 等连续生成方法也很重要。

这类方法可以粗略理解为：学习一条从简单分布到真实数据分布的连续变换路径。

```text
噪声分布
  -> 学习连续变换方向
  -> 数据分布
```

它们关注的问题包括：

- 如何减少采样步数。
- 如何让生成路径更直接。
- 如何提升训练和采样效率。
- 如何统一看待扩散、流和连续生成过程。

这类方法的具体实现会继续变化，但它们背后的长期问题很稳定：如何用更少步骤、更低成本、更稳定地从简单随机变量生成复杂内容。

## 文本条件：Prompt 不是咒语

文生图和文生视频最常见的控制方式是文本 Prompt。

但 Prompt 不是魔法咒语，它本质上是条件信息。

```text
文本条件
  -> 文本编码器
  -> 条件向量 / Cross-Attention
  -> 影响生成过程
```

一个好的生成系统要解决的不只是“听懂关键词”，还包括：

- 主体是否准确。
- 风格是否一致。
- 空间关系是否可靠。
- 数量和文字是否正确。
- 细节是否符合用户意图。
- 负面约束是否生效。

Prompt 工程在短期内会随模型变化而变化，但“条件表达是否清晰、约束是否可执行、结果是否可验证”这三个问题长期存在。

## 图像条件：从参考到控制

图像条件让生成不再只依赖文本。

常见图像条件包括：

| 条件类型 | 作用 |
| --- | --- |
| 参考图 | 提供风格、人物、物体或场景参考 |
| 边缘图 | 控制轮廓结构 |
| 深度图 | 控制空间和前后关系 |
| 姿态图 | 控制人物动作 |
| 分割图 | 控制区域布局 |
| 草图 | 提供粗略构图 |
| Mask | 指定编辑区域 |
| 首帧 | 控制视频起点 |

这类条件的价值是把用户意图从“自然语言描述”变成更明确的结构约束。

文本适合表达语义，图像条件适合表达空间、形状、布局和身份一致性。

## ControlNet：可控生成的代表思路

ControlNet 的核心价值在于：让扩散模型在不破坏原有生成能力的情况下，接入额外结构条件。

例如：

```text
文本 Prompt + 边缘图
  -> 生成符合边缘结构的图像

文本 Prompt + 姿态图
  -> 生成符合人物姿态的图像
```

它代表的不只是一个具体模型，而是一类重要思想：

> 生成模型要从“凭描述生成”走向“按约束生成”。

对真实创作和生产来说，可控性往往比单次惊艳更重要。

因为用户通常不是想要“随便一张好看的图”，而是想要“符合这个构图、这个人物、这个产品、这个镜头、这个版式的图”。

## 图像编辑：生成模型的实用核心

图像编辑比纯文生图更接近真实工作流。

典型编辑任务包括：

- 局部重绘
- 背景替换
- 风格迁移
- 物体增删
- 属性修改
- 图像扩展
- 超分辨率
- 旧图修复
- 文本指导编辑

编辑任务的难点在于平衡“改”和“不改”。

```text
应该改变：
  用户指定的区域、风格或对象

应该保持：
  身份、构图、未编辑区域、光照、透视、整体一致性
```

很多生成模型失败，不是因为生成质量差，而是因为它没有尊重编辑边界。

这也是图像编辑长期重要的原因：它要求模型既有创造力，也有约束感。

## 身份一致性与角色一致性

在人像、IP 角色、商品图、品牌视觉和视频生成中，一致性是核心问题。

用户希望模型在不同姿态、场景、镜头和时间中保持同一个人、同一个角色或同一个产品。

这涉及：

- 参考图编码
- 个性化微调
- 文本反演
- DreamBooth 类方法
- LoRA / Adapter
- 角色特征绑定
- 多图一致性约束

长期来看，生成模型从“生成一张图”走向“生成一个可复用的视觉主体”，是内容生产系统化的关键。

## 视频生成：多了时间维度

视频生成可以看作图像生成加上时间一致性，但实际难很多。

```text
图像生成：
  空间一致性

视频生成：
  空间一致性 + 时间一致性 + 运动合理性
```

视频生成需要处理：

- 主体在帧间保持一致。
- 运动轨迹合理。
- 镜头变化自然。
- 场景和光照连续。
- 物理关系不突变。
- 长视频中叙事和动作不漂移。

扩散模型、自回归模型和视频 Transformer 都可以用于视频生成。无论路线怎么变，视频生成长期绕不开三个问题：

- **一致性**：同一个主体不能每帧变样。
- **可控性**：用户要控制镜头、动作、节奏和风格。
- **长程结构**：视频不是独立帧拼接，而是有时间组织的内容。

这一部分会在后续 [视频多模态](../07_视频多模态/README.md) 章节展开。

## 音频与语音生成

音频生成和图像生成不同，但底层问题有相似之处：都要从条件生成连续信号。

常见任务包括：

- 文本转语音
- 声音克隆
- 音乐生成
- 音效生成
- 语音转换
- 音频修复
- 视频配音和配乐

音频生成更强调时间连续性、节奏、音色和可听自然度。

与图像相比，音频结果更难通过单帧检查，需要从时间轴上听整体效果。节奏、停顿、情绪、发音、噪声和混响都会影响体验。

## 统一多模态生成

统一多模态生成希望一个模型或系统能同时处理多种输入和输出。

理想形态可能是：

```text
输入：
  文本 + 图像 + 音频 + 视频片段 + 用户指令

输出：
  文本回答 + 编辑图像 + 新视频 + 语音说明
```

它的难点不只是模型规模，而是不同模态的表示、时间尺度和评价标准都不一样。

| 模态 | 表示特点 | 生成难点 |
| --- | --- | --- |
| 文本 | 离散 Token | 逻辑、事实、风格、长上下文 |
| 图像 | 二维空间结构 | 真实感、布局、细节、一致性 |
| 视频 | 空间 + 时间 | 运动、主体一致、长程叙事 |
| 音频 | 连续时间信号 | 音质、节奏、音色、情绪 |
| 3D | 几何和材质 | 形状、视角一致、可编辑 |

统一生成模型的长期挑战，是让这些模态在同一个系统中协调，而不是简单把多个模型串起来。

## 多模态生成模型的训练数据

生成模型很吃数据。

不同任务需要不同数据：

| 数据类型 | 作用 |
| --- | --- |
| 图文对 | 学习文本和视觉内容对应关系 |
| 高质量图片 | 学习视觉质量、构图、风格和细节 |
| 图像编辑对 | 学习如何根据指令修改图像 |
| 视频文本对 | 学习动作、镜头、事件和时间关系 |
| 音频文本对 | 学习声音和语言描述对应关系 |
| 多图一致性数据 | 学习角色、产品、风格一致性 |
| 人类偏好数据 | 学习审美、可用性和安全边界 |

长期来看，数据问题会比模型结构更顽固。

原因很简单：生成任务没有唯一答案。什么是“好图”“好视频”“好声音”，往往依赖语境、审美、用途和用户意图。

## 生成模型的评测

生成模型评测很难，因为生成结果不只有一个标准答案。

常见评测维度包括：

| 维度 | 核心问题 |
| --- | --- |
| 视觉质量 | 是否清晰、自然、有细节？ |
| 文本对齐 | 是否符合 Prompt？ |
| 结构正确 | 数量、位置、关系是否正确？ |
| 可控性 | 控制条件是否生效？ |
| 一致性 | 人物、物体、风格是否稳定？ |
| 多样性 | 是否能生成不同合理结果？ |
| 编辑保真 | 该保留的内容是否保留？ |
| 时间稳定 | 视频帧间是否连贯？ |
| 安全性 | 是否生成有害、侵权或敏感内容？ |
| 可用性 | 是否能直接进入真实工作流？ |

单个分数很难说明全部问题。

一个更可靠的评测方式是：

```text
自动指标
  + 人工偏好
  + 任务集
  + 失败案例
  + 真实工作流测试
```

尤其在商业和创作场景里，“能否稳定完成用户指定任务”往往比单张样例是否惊艳更重要。

## 生成模型的常见失败模式

### 1. 语义跑偏

模型生成了高质量内容，但没有遵循用户意图。

例如用户要求“两只狗”，结果生成三只；要求“红色杯子在桌子左侧”，结果位置反了。

### 2. 结构错误

手指数量、文字、几何关系、透视、物体交互经常出错。

这说明生成质量和结构理解不是一回事。

### 3. 局部好，全局乱

局部纹理很漂亮，但整体构图、空间关系或故事逻辑不成立。

### 4. 编辑越界

用户只想改背景，模型却把人物脸、衣服、光照也改了。

这是编辑模型最常见的可用性问题之一。

### 5. 一致性不足

同一个人物或产品在多张图、多帧视频中发生漂移。

### 6. 文本生成困难

图像中的文字、Logo、表格、公式仍然是许多生成模型的难点。

### 7. 安全和版权风险

生成模型可能涉及肖像、商标、版权风格、敏感内容和误导性合成。

越接近真实生产，这些问题越不能忽略。

## 工程视角：生成模型不是一个按钮

真实生成系统通常不是单一模型。

一个实用的内容生成工作流可能包含：

```text
意图理解
  -> Prompt 改写
  -> 条件解析
  -> 生成
  -> 自动筛选
  -> 局部编辑
  -> 超分辨率
  -> 安全审核
  -> 导出和版本管理
```

其中每一步都影响最终体验。

很多时候，产品差距不只来自底层模型，还来自：

- 是否能把用户意图转成有效条件。
- 是否支持可回退、可局部修改。
- 是否能保留角色和品牌一致性。
- 是否能处理失败样例。
- 是否能在成本和延迟内完成任务。

所以学习多模态生成，不能只看模型论文，也要看系统工作流。

## 如何阅读多模态生成论文？

读一篇生成模型论文，可以按下面几个问题拆解：

1. **生成对象是什么？** 图像、视频、音频、3D，还是统一多模态？
2. **表示空间是什么？** 像素、离散 Token、连续 Latent，还是混合表示？
3. **生成范式是什么？** 扩散、自回归、掩码生成、流匹配，还是组合方案？
4. **条件如何接入？** 文本、图像、边缘、姿态、Mask、参考图、音频是否参与控制？
5. **训练数据是什么？** 数据规模、质量、过滤方式和任务覆盖如何？
6. **采样如何完成？** 步数、速度、分辨率、成本和可复现性如何？
7. **可控性如何验证？** 是否只展示好看的样例，还是有系统评测？
8. **失败案例是什么？** 是否讨论结构错误、一致性、编辑越界和安全问题？
9. **能否进入工作流？** 是否支持编辑、迭代、批量生成、风格保持和版本管理？

这套问题适用于大多数生成模型论文，也适合判断一个生成产品是否真正可用。

## 长期不变的判断框架

生成模型更新很快，但下面这些问题会长期存在。

### 1. 表示是否合适？

不同模态需要不同表示。表示空间决定了模型能否高效学习，也决定了最终质量上限。

### 2. 条件是否清楚？

用户意图越复杂，越需要结构化条件。只靠文本 Prompt 很难稳定控制布局、身份、动作和细节。

### 3. 生成是否可控？

真实生产不是抽卡。用户需要可预测、可调整、可复用的结果。

### 4. 编辑是否可靠？

创作工作流通常是迭代式的。能不能局部修改、保留关键内容，比一次生成更重要。

### 5. 一致性是否稳定？

角色、产品、品牌、场景、镜头和时间的一致性，是生成模型走向专业生产的门槛。

### 6. 评测是否贴近任务？

好看的样例不等于可靠系统。评测必须覆盖真实任务和失败案例。

### 7. 系统是否闭环？

生成只是一步。理解、规划、生成、编辑、审核、导出、反馈共同决定产品价值。

## 常见误区

### 1. 多模态生成不等于文生图

文生图只是入口。图像编辑、视频生成、音频生成、3D 生成和统一多模态生成都属于更大的生成体系。

### 2. 扩散模型不是唯一答案

扩散模型很强，但自回归、掩码生成、流匹配等路线同样重要。不同任务可能适合不同范式。

### 3. Prompt 不是全部控制力

复杂控制通常需要图像条件、结构条件、参考图、Mask、时间轴和工作流协同。

### 4. 清晰不等于正确

一张图很清晰，也可能语义错误、结构错误、人物不一致或不符合编辑指令。

### 5. 生成模型不是越自由越好

在创作产品中，约束和可编辑性往往比随机创造更有价值。

### 6. 统一模型不等于简单拼接

把多个模态塞进一个模型，不等于真正统一。统一需要表示、训练目标、上下文、采样和评测共同设计。

## 本章小结

多模态生成模型的核心可以概括为：

```text
选择合适的内容表示
  -> 选择生成范式
  -> 接入文本、图像、结构和时间条件
  -> 生成可用内容
  -> 支持编辑、控制、评测和安全闭环
```

几年之后，具体模型名字可能会变，但这些问题不会过时：

- 内容如何表示？
- 意图如何表达？
- 生成如何采样？
- 条件如何控制？
- 结果如何编辑？
- 一致性如何保持？
- 质量如何评测？
- 系统如何落地？

这也是学习多模态生成时最值得抓住的主线。
