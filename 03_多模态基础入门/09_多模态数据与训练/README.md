# 09_多模态数据与训练

> 多模态模型的能力不只来自架构和参数规模，更来自数据如何采集、清洗、配对、混合、训练和评测。数据决定模型看过什么，也决定它容易忽略什么、误解什么和编造什么。

**By：猫先生 of 「魔方AI空间」**

## 本章导读

前面几章分别讨论了图像、视频、音频如何进入模型，以及模型如何理解和生成多模态内容。

但把视觉编码器、连接器和 LLM 拼在一起，并不会自然得到一个可靠的多模态模型。模型还需要通过数据和训练学会：

- 不同模态之间如何对应。
- 什么视觉信息值得关注。
- 如何根据指令使用图像、视频或音频。
- 看不清、听不清时如何表达不确定。
- 如何避免依赖语言先验编造答案。
- 如何适应文档、图表、OCR、GUI、视频和真实业务场景。

一个简化训练流程是：

```text
原始多模态数据
  -> 授权与治理
  -> 清洗、过滤、去重
  -> 配对与标注
  -> 数据混合与采样
  -> 多模态预训练
  -> 指令微调
  -> 偏好与安全对齐
  -> 评测和失败样例回流
```

本章重点回答：

- 图文对、交错图文、指令数据和偏好数据有什么区别？
- 多模态训练为什么常常分阶段进行？
- 对比学习、生成式训练、指令微调和偏好优化分别学什么？
- 数据质量、规模和多样性应该如何权衡？
- 合成数据有什么价值，又会带来什么风险？
- 多模态幻觉、评测污染和数据泄漏为什么首先是数据问题？
- 如何建立几年后仍然有效的数据与训练判断框架？

## 一句话理解多模态训练

多模态训练可以理解为：

> 用不同粒度、不同目标的数据，逐步教模型建立跨模态对应关系，并学会在真实任务中可靠地使用这些关系。

它不是一次训练完成，而更像一套课程：

```text
先认识图像和文本之间的关系
  -> 再学会把视觉特征接入语言模型
  -> 再学习遵循多模态指令
  -> 再学习人类偏好、安全边界和业务规范
```

模型结构提供能力容器，数据和训练决定这个容器里最终装了什么。

## 数据不是燃料，而是行为边界

把数据比作“燃料”不够准确。燃料只影响机器能不能运行，而训练数据还会改变模型的行为。

数据会影响：

- 模型认识哪些对象、语言、文化和场景。
- 模型是否擅长 OCR、图表、文档或视频。
- 模型会不会忽略细粒度视觉证据。
- 模型回答问题时更依赖图像还是语言先验。
- 模型如何处理模糊、冲突和高风险信息。
- 模型对不同群体、地区和设备条件是否公平。

因此，多模态数据治理不是训练前的杂务，而是模型设计的一部分。

## 多模态数据的基本类型

不同数据教给模型的能力不同。

| 数据类型 | 典型形式 | 主要作用 |
| --- | --- | --- |
| 图文对 | 图片 + Caption / alt text | 学习图文语义对齐 |
| 交错图文 | 文档或网页中的多图多段文本 | 学习图文上下文和多图关系 |
| 区域标注 | 图像区域 + 类别 / 短语 / 框 | 学习定位和细粒度对齐 |
| OCR / 文档 | 页面图像 + 文字 + 版面结构 | 学习文字、表格和布局 |
| 图表数据 | 图表 + 数据 + 问题答案 | 学习数值和结构推理 |
| 视频文本 | 视频片段 + Caption / 字幕 | 学习动作、事件和时间关系 |
| 音频文本 | 音频 + 转写 / 描述 | 学习语音和声音语义 |
| 多模态指令 | 模态输入 + 用户问题 + 回答 | 学习按指令完成任务 |
| 偏好数据 | 多个回答 + 人类或模型偏好 | 学习质量、安全和交互偏好 |
| 工具轨迹 | 观察 + 动作 + 反馈 | 学习 GUI、Agent 和工具调用 |

数据类型不能相互完全替代。

大量图文对可以让模型认识视觉概念，但不一定教会它进行多轮问答；大量问答数据可以改善指令遵循，但不一定补足底层视觉表示。

## 图文对数据

图文对是多模态预训练最常见的数据形态。

```text
image
  <-> caption / title / alt text
```

图文对可以来自：

- 人工标注数据集。
- 网页图片和 alt text。
- 商品图和商品描述。
- 新闻图片和图注。
- 视频帧和字幕。
- 模型生成的 Caption。

它们主要用于学习全局语义对齐：

```text
“一只黑猫坐在窗台上”
  <-> 对应图片
```

但图文对经常存在弱对齐问题：

- 文本描述网页，而不是图片。
- Caption 只描述主体，没有提到背景和细节。
- 图片中有文字，Caption 没有转写。
- 图文包含错误、广告、模板或无关信息。
- 同一图片被不同低质量文本重复配对。

所以图文对的规模重要，图文是否真的对应更重要。

## 交错图文数据

交错图文数据不是一张图配一句话，而是在同一上下文中包含多张图和多段文字。

```text
段落 1
图片 A
段落 2
图片 B
段落 3
```

这类数据常见于：

- 网页
- 教材
- 技术文档
- 新闻
- 博客
- 说明书
- 演示文稿
- 科研论文

交错图文数据可以教模型：

- 区分多张图。
- 理解图片和前后文本的关系。
- 处理多图上下文。
- 学习文档结构。
- 在长上下文中引用视觉信息。

它比简单图文对更接近真实知识载体，但清洗、切分和归属判断也更复杂。

## 区域级和细粒度数据

全局 Caption 告诉模型“图片大概是什么”，区域标注则告诉模型“具体在哪里”。

区域级数据可能包含：

| 标注 | 作用 |
| --- | --- |
| Bounding Box | 指定对象位置 |
| Segmentation Mask | 指定精确区域 |
| Region Caption | 描述局部区域 |
| Referring Expression | 用语言指向特定对象 |
| Keypoint / Pose | 表示人体或物体结构 |
| OCR Box | 关联文字和位置 |

这类数据对以下能力尤其重要：

- Grounding
- 对象定位
- GUI 操作
- 文档理解
- 细粒度问答
- 视觉 Agent

如果训练数据只有整图描述，模型可能知道“图里有什么”，却不知道“它在哪里”。

## OCR、文档和图表数据

OCR 和文档理解不能只靠普通自然图像数据解决。

文档场景包含：

- 小字
- 表格
- 多栏排版
- 页眉页脚
- 脚注
- 公式
- 图表
- 印章和手写内容
- 跨页引用

文档数据不仅要有文字转写，还要保留布局和结构。

```text
页面图像
  + OCR 文本
  + 文本框坐标
  + 阅读顺序
  + 表格结构
  + 问答和引用位置
```

图表数据还需要区分：

- 视觉元素
- 坐标轴
- 图例
- 数值
- 数据关系
- 推理问题

模型能读文字，不代表它能理解文档；能理解文档，也不代表它能准确进行数值推理。

## 视频和音频数据

视频、音频数据比图文对多了时间维度。

| 数据 | 需要关注的对齐关系 |
| --- | --- |
| 视频 + Caption | 描述是否覆盖真实片段 |
| 视频 + 字幕 | 字幕是否和说话时间同步 |
| 视频 + 动作标签 | 动作的起止时间是否准确 |
| 音频 + 转写 | 说话内容、说话人和时间戳是否对应 |
| 音频 + 描述 | 描述是否包含环境声和事件 |
| 音视频联合数据 | 声音是否来自画面中的事件 |

如果时间戳不准，模型会学到错误的因果和顺序。

所以视频和音频数据不仅要“配对”，还要“在时间上配准”。

## 多模态指令数据

多模态指令数据通常包含：

```text
图像 / 视频 / 音频
  + 用户指令
  + 助手回答
```

它和普通图文对的区别在于：

| 图文对 | 多模态指令数据 |
| --- | --- |
| 学习图像和文本是否对应 | 学习如何根据用户意图使用图像 |
| 常用于预训练和检索 | 常用于 SFT 和助手对齐 |
| 文本多为描述 | 文本包含问题、命令、约束和回答 |
| 强调语义空间 | 强调任务完成和交互行为 |

指令数据可以覆盖：

- 图像描述
- 视觉问答
- OCR 问答
- 文档抽取
- 图表分析
- 多图比较
- 视频摘要
- 音频理解
- 视觉推理
- 工具调用
- 拒答与不确定性表达

视觉指令微调让模型从“能接收视觉信息”走向“知道如何使用视觉信息”。

## 多模态偏好数据

指令微调告诉模型怎样回答，偏好数据进一步告诉模型什么回答更好。

一个偏好样本可能包含：

```text
图像 + 问题
  -> 回答 A
  -> 回答 B
  -> 偏好标签：A 优于 B
```

偏好维度可以包括：

- 是否忠于视觉证据。
- 是否完整回答问题。
- 是否简洁清楚。
- 是否避免编造。
- 是否说明不确定性。
- 是否遵守安全要求。
- 是否给出正确引用和位置。

多模态偏好数据比纯文本偏好数据更难，因为标注者必须同时检查输入模态和回答。

## 多模态数据的生命周期

一套可持续的数据体系通常包括以下环节：

```text
定义能力目标
  -> 设计数据规范
  -> 获取数据
  -> 权利和安全审查
  -> 解码与格式标准化
  -> 清洗、过滤、去重
  -> 配对、标注和质量评分
  -> 混合与采样
  -> 训练
  -> 评测
  -> 失败样例回流
  -> 数据版本迭代
```

如果没有生命周期管理，数据集很容易变成无法解释、无法复现、无法更新的黑箱。

## 先定义能力，再收集数据

“先找尽可能多的数据，再看能训练出什么”不是唯一方式，也往往不是最有效的方式。

更稳健的做法是先明确能力地图：

| 能力 | 需要的数据 |
| --- | --- |
| 通用视觉语义 | 大规模图文对 |
| OCR | 多字体、多语言、多场景文字图像 |
| 文档理解 | 页面、布局、表格、阅读顺序、问答 |
| 图表推理 | 图表、底层数据、问题、计算过程 |
| 多图比较 | 成组图像和跨图问题 |
| 视频理解 | 片段、时间戳、事件、字幕、问答 |
| 音频理解 | 语音、环境声、转写、事件描述 |
| GUI Agent | 截图、控件、动作、状态变化 |
| 幻觉控制 | 无答案、证据不足、反事实和冲突样本 |

能力目标越清楚，数据缺口越容易被发现。

## 数据授权与治理

多模态数据通常包含更复杂的权利问题：

- 图片和视频版权
- 人物肖像
- 声音和声纹
- 地理位置
- 医疗和身份信息
- 商标和品牌资产
- 未成年人内容
- 私密场景
- 生成内容的来源和许可

数据治理至少需要回答：

1. 数据从哪里来？
2. 是否有训练和再分发权利？
3. 是否包含个人敏感信息？
4. 是否提供删除或退出机制？
5. 数据是否跨地区、跨语言合规？
6. 模型生成结果可能复现哪些训练内容？

这些问题不会因为模型更强而消失，反而会随着模型能力增强变得更重要。

## 数据清洗

多模态数据清洗通常包含多个层次。

### 1. 文件和格式清洗

- 图片、视频、音频能否正常解码。
- 分辨率、时长、采样率是否合理。
- 文件是否损坏。
- 色彩空间和方向信息是否正确。
- 动图、透明图、缩略图是否需要特殊处理。

### 2. 内容质量过滤

- 图像是否过度模糊。
- 视频是否几乎静止或重复。
- 音频是否全是静音或噪声。
- 文本是否乱码、过短、模板化。
- 内容是否被水印、边框、拼图严重干扰。

### 3. 语义相关性过滤

- 图像和文本是否真的相关。
- 视频 Caption 是否描述对应片段。
- 音频转写是否和声音一致。
- 问题是否能从输入中回答。
- 答案是否得到视觉或听觉证据支持。

### 4. 安全和政策过滤

- 违法和高风险内容
- 隐私和个人信息
- 仇恨、暴力和性内容
- 未授权肖像和声音
- 可能被用于危险能力的数据

清洗不是越严格越好。过度过滤会损失长尾知识、真实噪声和边界场景，导致模型只会处理“漂亮的标准样例”。

## 图文相关性不等于信息充分

一张狗的图片配上“动物”这句话，图文是相关的，但信息很弱。

高质量配对通常需要考虑：

| 维度 | 问题 |
| --- | --- |
| 相关性 | 文本是否描述这张图？ |
| 具体性 | 是否包含主体、属性、动作和关系？ |
| 覆盖度 | 是否遗漏关键内容？ |
| 正确性 | 是否包含错误或臆测？ |
| 可用性 | 是否适合目标训练任务？ |

因此，数据质量评分不应该只有一个“相似度分数”，而要结合任务目标。

## 去重

多模态去重至少包含几种层次。

| 去重类型 | 说明 |
| --- | --- |
| 文件级去重 | 完全相同的文件 |
| 感知去重 | 尺寸、压缩或轻微裁剪不同，但内容相同 |
| 语义去重 | 内容和语义高度重复 |
| 模板去重 | 同一页面模板、商品模板或字幕模板大量重复 |
| 跨集合去重 | 训练集和验证集、测试集之间去重 |

重复数据会造成：

- 热门样本被过度学习。
- 长尾数据被淹没。
- 训练成本浪费。
- 模型记忆特定内容。
- Benchmark 污染。
- 评测结果虚高。

去重不是只为了节省空间，而是为了让数据分布更真实。

## 训练集污染与评测泄漏

多模态 Benchmark 往往公开在互联网，题目、图片和答案可能被训练数据收录。

污染形式包括：

- 测试图片出现在训练集中。
- 测试问题和答案出现在网页中。
- 图片经过裁剪、压缩或加水印后重复出现。
- Benchmark 解题文章被抓取。
- 合成数据模型已经见过测试集。

因此，评测去污染需要：

- 图像感知哈希。
- 文本近似匹配。
- 语义检索。
- 数据来源追踪。
- 时间切分。
- 保密测试集。
- 动态或新生成评测。

如果不知道模型是否见过测试题，Benchmark 分数就很难解释。

## 数据分布与长尾

互联网数据并不等于真实世界的均匀采样。

常见偏差包括：

- 热门语言过多。
- 西方互联网内容过多。
- 高质量摄影图过多，普通手机图不足。
- 正面、中心主体图片过多。
- 网红、广告、商品和模板内容重复。
- 罕见场景、弱光、低清晰度样本不足。
- 无障碍、老年、儿童和特殊设备场景不足。

数据分布会直接变成模型能力分布。

如果训练数据很少包含模糊截图，模型就不擅长真实办公截图；如果很少包含小语种文档，模型就会在这些语言上表现不稳定。

## 数据配比

多模态训练通常会混合多种数据。

```text
通用图文
  + OCR / 文档
  + 图表
  + 多图
  + 视频
  + 音频
  + 指令
  + 安全数据
```

配比决定模型把容量花在哪里。

常见取舍：

- 通用数据过多，专项能力可能不足。
- OCR 数据过多，模型可能偏向读字而忽略视觉语义。
- 合成问答过多，回答风格可能模板化。
- 单图数据过多，多图和视频能力会弱。
- 高质量小数据占比过低，可能被噪声数据淹没。

数据配比不是一次决定，而需要通过评测不断调整。

## 采样策略

直接按数据量随机采样，往往会让大数据源支配训练。

常见采样策略包括：

| 策略 | 作用 |
| --- | --- |
| 按数据源加权 | 控制不同来源的占比 |
| 温度采样 | 减弱大数据源的支配效应 |
| 能力均衡采样 | 按任务能力分桶 |
| 难度采样 | 逐步加入更难样本 |
| 质量加权 | 提高高质量样本被采到的概率 |
| 在线重采样 | 根据训练状态动态调整 |

采样策略本质上是在定义模型的学习课程。

## 多模态训练为什么分阶段？

多模态模型通常复用已有视觉编码器和 LLM。不同模块的表示空间并不天然兼容。

因此训练常分为多个阶段：

```text
阶段 1：单模态或跨模态基础预训练
阶段 2：视觉 / 音频特征与 LLM 对齐
阶段 3：多模态联合预训练
阶段 4：多模态指令微调
阶段 5：偏好与安全对齐
阶段 6：领域适配和持续训练
```

分阶段的好处：

- 降低训练难度。
- 复用已有模型能力。
- 控制不同模块更新幅度。
- 更容易定位问题。
- 避免早期噪声破坏 LLM 能力。

但分阶段也可能产生目标不一致和能力遗忘，需要通过数据回放、混合训练和联合优化缓解。

## 阶段一：基础表示与跨模态预训练

这一阶段的目标是学习模态内表示和模态间对应关系。

常见任务：

- 图文对比学习
- 图文匹配
- 图像 Caption
- 掩码语言或视觉建模
- 音频文本对齐
- 视频文本对齐
- 视觉或音频自监督学习

CLIP、ALIGN、BLIP、CLAP 等工作体现了不同的跨模态预训练思路。

这一阶段通常依赖大规模数据，重点是覆盖和表示能力。

## 阶段二：连接器对齐

当视觉编码器和 LLM 已经存在时，可以先冻结大部分参数，只训练连接器。

```text
冻结视觉编码器
冻结 LLM
训练 Projector / Q-Former / Resampler
```

目标是让视觉特征进入 LLM 的表示空间。

优点：

- 训练成本低。
- 不容易破坏已有单模态能力。
- 适合快速验证架构。

局限：

- 连接器容量可能成为瓶颈。
- 视觉编码器和 LLM 无法共同适配。
- 对细粒度、复杂推理和新模态支持有限。

## 阶段三：联合多模态训练

联合训练会解冻部分或全部模块，让视觉编码器、连接器和 LLM 一起适配。

可以选择：

| 训练范围 | 特点 |
| --- | --- |
| 只训练连接器 | 成本最低，能力上限受限 |
| 连接器 + LLM | 改善视觉信息使用方式 |
| 连接器 + 视觉编码器 | 改善专项视觉表示 |
| 全参数训练 | 适配能力强，成本和风险最高 |
| Adapter / LoRA | 在成本和可塑性之间折中 |

是否解冻，不存在对所有任务都最优的答案。

需要考虑：

- 数据规模和质量。
- 计算预算。
- 是否需要保留通用语言能力。
- 是否有足够专项数据。
- 模型是否出现灾难性遗忘。

## 阶段四：多模态指令微调

多模态 SFT 教模型理解用户指令和对话格式。

训练样本通常是：

```text
system prompt
  + image / video / audio
  + user instruction
  + assistant response
```

它重点改善：

- 指令遵循
- 多轮对话
- 输出格式
- 任务迁移
- 多图引用
- 不确定性表达
- 工具调用

指令微调数据规模不一定需要最大，但质量、覆盖和任务格式很重要。

## 阶段五：偏好与安全对齐

SFT 回答“应该怎样答”，偏好优化回答“多个可行答案里哪个更好”。

常见方法包括：

- RLHF
- RLAIF
- DPO
- 规则和拒答数据
- 模型批评与自我修正

多模态偏好对齐尤其需要关注：

- 回答是否忠于输入模态。
- 模型是否在看不清时继续猜。
- 是否错误引用图像区域或视频时间。
- 是否处理敏感图像、声音和人物身份。
- 是否遵守业务场景输出规范。

偏好对齐不能替代底层视觉能力。如果模型根本没看清，优化回答风格无法让答案变正确。

## 训练目标

多模态训练常见目标可以分为几类。

| 目标 | 学到什么 |
| --- | --- |
| 对比学习 | 匹配模态在语义空间中靠近 |
| 图文匹配 | 判断图像和文本是否对应 |
| Caption / 自回归生成 | 根据模态输入生成文本 |
| 掩码建模 | 从上下文恢复被遮挡内容 |
| 重建目标 | 保留像素、音频或潜变量信息 |
| 下一 Token 预测 | 在统一序列中生成文本或模态 Token |
| 定位目标 | 学习区域、坐标和时间戳 |
| 偏好目标 | 提高人类更偏好的回答概率 |
| 安全目标 | 学习拒答、边界和风险处理 |

不同目标会塑造不同能力。

只做对比学习，模型可能适合检索但不擅长生成；只做下一 Token 预测，模型可能语言流畅但视觉定位不足。

## 对比学习

对比学习让匹配的跨模态样本靠近，不匹配样本远离。

```text
匹配图文对：
  image A <-> caption A

负样本：
  image A <-> caption B
```

对比学习的关键不只是正样本，还包括负样本。

如果负样本太容易，模型只需学习粗粒度类别；如果负样本具有细微差异，模型才会学习属性、关系和细节。

例如：

```text
正样本：红色杯子在蓝色书本左侧
困难负样本：蓝色杯子在红色书本左侧
```

困难负样本和反事实样本，对细粒度视觉理解非常重要。

## 生成式训练

生成式训练让模型根据图像、视频或音频生成文本或模态 Token。

常见任务：

- 图像描述
- 视频 Caption
- 音频描述
- OCR 转写
- 文档问答
- 图像或音频 Token 生成

生成式目标能让模型学习更丰富的条件分布，但也容易出现语言先验过强的问题。

模型可能生成“听起来合理”的回答，却没有充分依据输入模态。

因此生成式训练需要和视觉 grounding、反事实数据、无答案样本和证据引用结合。

## 课程学习

多模态训练可以看作课程设计。

一个常见课程是：

```text
清晰单图和简单 Caption
  -> 多对象和关系
  -> OCR、图表、文档
  -> 多图、视频、音频
  -> 推理、工具和真实业务任务
```

课程学习的价值是让模型从稳定基础逐步过渡到复杂任务。

但“先简单后困难”不是唯一策略。更重要的是训练过程是否保持能力平衡，是否及时暴露模型薄弱点。

## 合成数据

合成数据是多模态训练的重要来源。

它可以用于：

- 为图片生成详细 Caption。
- 为文档生成问答。
- 为图表生成推理题。
- 为视频生成摘要和时间问题。
- 生成反事实和困难负样本。
- 扩充小语种和长尾场景。
- 生成拒答、安全和格式约束样本。

合成数据的优势：

- 成本较低。
- 容易控制任务格式。
- 能补足稀缺能力。
- 能定向覆盖失败案例。

风险：

- 教师模型错误被放大。
- 文风和答案模板趋同。
- 数据多样性被高估。
- 训练模型继承教师偏见。
- 合成样本看似复杂，实际缺少真实感。
- 教师模型可能已经受测试集污染。

合成数据不是“免费高质量数据”，而是一种需要验证和审计的数据来源。

## 自训练与数据飞轮

模型可以参与自己的数据生产：

```text
当前模型
  -> 在未标注数据上生成候选结果
  -> 过滤、验证和人工抽查
  -> 形成新训练数据
  -> 训练下一版本模型
```

这就是常说的数据飞轮。

有效飞轮需要：

- 可靠的质量过滤器。
- 独立于生成模型的验证信号。
- 人工审查关键样本。
- 真实用户失败案例。
- 数据版本和来源记录。

如果生成、筛选和评测都由同一种模型完成，系统容易形成自我确认偏差。

## 多模态幻觉与数据

多模态幻觉常被归因于模型，但它也和数据强相关。

可能原因：

- Caption 经常补充图中没有的信息。
- 问答数据鼓励模型必须给出答案。
- 缺少“看不清”“无法判断”的样本。
- 文本先验比视觉证据更稳定。
- 训练数据很少包含反事实和冲突输入。
- 评价标准偏好流畅完整，而非谨慎准确。

改善方向：

- 加入无答案和证据不足样本。
- 加入视觉反事实数据。
- 加入局部证据和坐标监督。
- 训练模型引用图像区域或视频时间戳。
- 奖励忠实性而不只奖励流畅度。
- 对错误自信回答进行偏好惩罚。

减少幻觉不是只加一句 Prompt，而是数据、训练目标和评测共同作用。

## 数据质量和数据规模

“质量还是规模”不是二选一。

| 数据 | 价值 |
| --- | --- |
| 大规模弱标注数据 | 提供覆盖、长尾和开放世界知识 |
| 高质量人工数据 | 提供精确任务行为和可靠监督 |
| 合成数据 | 定向补能力缺口和稀缺场景 |
| 真实用户数据 | 暴露系统失败和分布外问题 |
| 困难样本 | 提升模型对边界情况的区分能力 |

更成熟的策略通常是分层使用：

```text
大规模数据学覆盖
高质量数据学行为
困难数据补边界
真实反馈修失败
```

规模决定模型能看到多少世界，质量决定它从世界中学到什么。

## 持续训练与灾难性遗忘

多模态模型会不断增加新能力：

- 新语言
- 新视觉领域
- 文档和 OCR
- 视频和音频
- 新工具
- 新安全规则

持续训练可能导致旧能力下降。

常见缓解方式：

- 混合回放旧数据。
- 保留通用文本数据。
- 对关键能力建立回归集。
- 控制学习率和解冻范围。
- 使用 Adapter 或 LoRA 隔离部分能力。
- 定期评估单模态和多模态能力。

新增能力不应该靠牺牲已有能力来换取，除非这是明确的产品取舍。

## 数据版本与可复现性

一个可维护的数据集应记录：

- 数据来源
- 授权状态
- 获取时间
- 清洗规则
- 去重规则
- 过滤模型和阈值
- 标注指南
- 数据配比
- 数据版本
- 已知偏差
- 与评测集的去污染方式

训练实验还应记录：

- 使用了哪个数据版本。
- 每个数据源采样多少。
- 训练阶段和顺序。
- 冻结或解冻哪些模块。
- 使用哪些目标和损失权重。
- 评测结果和失败样例。

没有数据版本，模型版本也很难真正复现。

## 评测驱动的数据迭代

数据和评测应该形成闭环。

```text
建立能力评测
  -> 发现失败模式
  -> 对失败样例分类
  -> 找到数据缺口
  -> 补充或重采样数据
  -> 重新训练
  -> 做回归评测
```

失败样例不应该只被记录为“模型答错”，还要追问：

- 是没看见，还是没理解？
- 是训练数据缺失，还是连接器丢失信息？
- 是语言先验压过了视觉证据吗？
- 是问题本身无法回答吗？
- 是评测答案不完整吗？
- 是领域分布发生变化吗？

只有把错误归因到正确环节，数据迭代才不会变成盲目加数据。

## 数据卡与训练卡

长期维护多模态项目，建议为数据和训练建立结构化文档。

数据卡至少包括：

| 项目 | 内容 |
| --- | --- |
| 用途 | 数据服务哪些能力和任务 |
| 来源 | 公开、授权、合成、人工或用户反馈 |
| 规模 | 样本数、模态、语言、时长和分辨率 |
| 处理 | 清洗、过滤、去重和标注流程 |
| 风险 | 偏差、隐私、版权、安全和已知缺陷 |
| 版本 | 创建时间、变更记录和负责人 |

训练卡至少包括：

| 项目 | 内容 |
| --- | --- |
| 模型组件 | 编码器、连接器、LLM、Tokenizer |
| 阶段 | 预训练、对齐、SFT、偏好和领域适配 |
| 参数更新 | 冻结、全参、LoRA 或 Adapter |
| 数据配比 | 各来源采样比例 |
| 训练配置 | Batch、学习率、上下文和分辨率 |
| 评测结果 | 通用、专项、安全和回归评测 |

这些记录看起来朴素，却是数据工程真正跨越周期的基础。

## 常见失败模式

### 1. 数据量很大，但图文不对齐

噪声配对会让模型学会忽略输入模态，依赖语言先验完成任务。

### 2. 指令数据很多，但任务高度重复

样本数量增长了，能力覆盖却没有增长。

### 3. 合成数据没有验证

教师模型错误、偏见和模板被批量复制。

### 4. 训练集和测试集污染

Benchmark 分数很高，但不能说明泛化能力。

### 5. 只补正确答案，不补拒答样本

模型学会“任何问题都要回答”，更容易产生幻觉。

### 6. 新能力训练破坏旧能力

增加视觉或视频数据后，语言能力、指令遵循或其他模态能力下降。

### 7. 数据配比靠直觉，缺少回归评测

某一数据源被放大后，模型行为悄悄偏移。

### 8. 数据没有版本和来源记录

出现问题时无法追踪，也无法复现实验。

## 工程视角：一个可维护的数据训练管线

真实项目可以把数据训练管线拆成：

```text
数据注册
  -> 权利与安全审查
  -> 格式标准化
  -> 质量和相关性评分
  -> 去重与去污染
  -> 标注和合成
  -> 数据版本发布
  -> 混合和采样
  -> 分阶段训练
  -> 自动评测与人工抽检
  -> 失败案例回流
```

每一步都应产生可审计记录。

一个可靠的数据平台不只负责“把文件送进训练”，还应该能回答：

- 某条样本来自哪里？
- 为什么被保留或删除？
- 哪个模型版本使用过它？
- 它是否和测试集重复？
- 哪类能力从这批数据中受益？
- 训练后哪些旧能力发生了回退？

## 与 LLM 训练章节的关系

多模态训练并没有推翻 LLM 的训练方法，而是在文本训练基础上增加了模态表示、跨模态对齐和证据忠实性问题。

| 相关章节 | 在多模态训练中的延伸 |
| --- | --- |
| [LLM 预训练](../../02_LLM基础入门/08_预训练/README.md) | 从文本 Token 扩展到图像、视频、音频特征或模态 Token |
| [指令微调](../../02_LLM基础入门/09_指令微调/README.md) | 从纯文本指令扩展到带模态输入的任务和对话 |
| [RLHF](../../02_LLM基础入门/10_RLHF/README.md) | 增加视觉忠实性、证据引用和多模态安全偏好 |

理解这些 LLM 训练基础，有助于区分哪些问题是通用训练问题，哪些问题是多模态特有问题。

## 如何阅读多模态训练论文？

读一篇多模态训练论文，可以按下面几个问题拆解：

1. **数据从哪里来？** 公开数据、网页数据、授权数据、合成数据还是私有数据？
2. **数据类型是什么？** 图文对、交错图文、指令、偏好、视频、音频还是工具轨迹？
3. **如何清洗？** 是否说明质量、相关性、去重、安全和隐私过滤？
4. **如何去污染？** 是否检查与 Benchmark 的重复？
5. **训练分几阶段？** 每个阶段学什么？
6. **哪些模块更新？** 编码器、连接器、LLM 是否冻结？
7. **目标函数是什么？** 对比、生成、匹配、定位还是偏好优化？
8. **数据如何配比？** 不同模态、语言和任务如何采样？
9. **合成数据如何验证？** 是否有独立过滤和人工抽查？
10. **是否报告能力回退？** 新能力提升时，旧能力是否下降？
11. **数据是否可追踪？** 是否提供数据卡、版本和限制说明？

如果论文只报告模型结构和最终分数，却几乎不谈数据，很多结论就很难复现和解释。

## 长期不变的判断框架

多模态训练方法会变化，但下面这些问题会长期存在。

### 1. 数据是否真的对齐？

文件放在一起不代表语义对齐，语义相关也不代表信息充分。

### 2. 数据是否覆盖目标能力？

模型不会凭空学会训练分布中长期缺失的任务。

### 3. 数据质量和规模是否分层使用？

大规模弱数据、高质量人工数据、合成数据和真实反馈各有作用。

### 4. 训练目标是否匹配最终任务？

检索、问答、定位、生成和 Agent 需要不同监督信号。

### 5. 模型是否学会尊重模态证据？

多模态模型不能只靠语言先验生成看似合理的答案。

### 6. 评测是否独立且无污染？

如果评测集出现在训练数据里，性能结论就失去可信度。

### 7. 数据是否可治理、可追踪、可删除？

来源、许可、隐私和版本记录是长期维护的底线。

### 8. 训练是否形成闭环？

评测、失败分析、数据补充和回归测试要持续循环。

## 常见误区

### 1. 数据越多一定越好

大量低质量、重复或错配数据可能浪费计算，并强化错误模式。

### 2. 图文相似度高就是高质量

相似度只能反映部分相关性，无法保证具体、完整和事实正确。

### 3. 合成数据可以完全替代真实数据

合成数据适合补缺口，但容易继承教师模型错误和模板。

### 4. 多模态 SFT 可以弥补底层视觉能力

如果编码器看不清或连接器丢失信息，SFT 很难凭空恢复细节。

### 5. 偏好对齐可以消除所有幻觉

偏好训练能改善行为，但无法替代可靠感知、证据监督和无答案数据。

### 6. Benchmark 高分证明训练有效

还需要排除污染、过拟合、提示敏感性和领域偏差。

### 7. 数据工程结束于训练启动

真正的数据工程还包括训练后归因、失败回流、版本管理和删除治理。

## 本章小结

多模态数据与训练的核心可以概括为：

```text
先定义能力
  -> 构建匹配的数据类型
  -> 做授权、清洗、去重和去污染
  -> 设计数据配比与采样
  -> 分阶段完成预训练、对齐、SFT 和偏好训练
  -> 用独立评测发现真实失败
  -> 让失败样例回到下一轮数据迭代
```

几年之后，模型架构、训练框架和数据规模都会变化，但这些问题不会过时：

- 数据来自哪里？
- 不同模态是否真正对齐？
- 数据覆盖了哪些能力，又遗漏了什么？
- 训练目标是否匹配最终任务？
- 模型是否尊重输入证据？
- 测试集是否被污染？
- 数据是否有权使用、可追踪和可治理？
- 新能力提升时，旧能力是否退化？

多模态模型最终学到的不是“数据文件”，而是数据背后的分布、偏差和行为规范。
