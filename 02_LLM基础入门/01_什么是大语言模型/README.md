# 01_什么是大语言模型

> 大语言模型不是“会说话的搜索引擎”，而是一类在海量文本上学习语言规律、知识关联和任务模式的基础模型。

## 本章导读

大语言模型（Large Language Model，LLM）是现代生成式 AI 的核心基础设施。它可以写作、对话、翻译、总结、写代码、做数学推理，也可以作为 Agent、RAG、多模态系统和自动化工具调用系统的大脑。

但从最底层看，LLM 的核心训练目标非常朴素：**根据已有上下文，预测下一个 Token**。

这一章先不深入复杂公式，而是回答几个最重要的问题：

- LLM 到底是什么？
- 它和传统 NLP 模型有什么区别？
- 为什么“预测下一个 Token”可以产生对话、写作和推理能力？
- GPT、BERT、T5、LLaMA 这些模型路线有什么差别？
- 什么是基础模型、对话模型、推理模型？

## 一句话理解 LLM

LLM 是一种基于深度神经网络的语言模型，它通过在海量文本数据上训练，学习语言中的统计规律、知识结构、表达方式和任务模式，从而能够根据上下文生成连贯且有用的文本。

更直白地说：

```text
输入一段文本上下文 -> 模型计算每个候选 Token 的概率 -> 选择下一个 Token -> 不断重复 -> 生成完整回答
```

例如：

```text
输入：人工智能的未来是
模型预测：开放的、多模态的、智能体化的……
```

模型并不是直接“查找答案”，而是在参数中压缩了训练语料里的大量模式，并在当前上下文条件下生成最可能、最合适的后续内容。

## 语言模型的基本任务

语言模型最核心的问题是：

> 给定前面的文本，后面最可能出现什么？

对于自回归语言模型，这个过程可以理解为：

```text
P(下一个 Token | 前面的所有 Token)
```

假设输入是：

```text
从零走向
```

模型可能会给出不同候选 Token 的概率：

| 候选 Token | 概率直觉 |
| --- | --- |
| AGI | 高 |
| 人工智能 | 中 |
| 未来 | 中 |
| 苹果 | 低 |

LLM 每次只预测一个或一小段 Token，但连续预测很多次后，就形成了完整句子、段落、代码甚至长篇文章。

## 为什么预测下一个 Token 会产生智能感？

“预测下一个 Token”听起来很简单，但当模型规模、数据规模和训练质量足够大时，这个任务会迫使模型学习很多隐藏能力。

### 1. 学习语言结构

为了预测下一个词，模型需要理解语法、词义、句子结构和上下文关系。

例如：

```text
我今天去了医院，因为我觉得身体有点
```

模型倾向于预测“不舒服”“发烧”“疼”等，而不是“蓝色”“汽车”。这说明它学到了语义和上下文关联。

### 2. 学习世界知识

训练语料中包含大量百科、论文、代码、新闻、教程和问答内容。为了预测文本，模型会把大量知识压缩进参数中。

例如：

```text
Transformer 是由 Google 在 2017 年论文
```

模型可能继续生成 “Attention Is All You Need 中提出的”。这不是数据库检索，而是参数化记忆与上下文推断的结果。

### 3. 学习任务模式

互联网上存在大量“问题 -> 答案”“代码注释 -> 代码”“文章标题 -> 正文”“英文 -> 中文”的文本模式。模型训练时会学习这些模式。

所以当我们输入：

```text
请把下面这句话翻译成英文：大语言模型正在改变软件开发。
```

模型能够识别这是一个翻译任务，并生成对应英文。

### 4. 产生涌现能力

当模型规模和训练数据达到一定水平后，一些小模型不明显的能力会变得更强，例如：

- 多步推理
- 代码生成
- Few-shot 学习
- 指令遵循
- 工具调用
- 复杂格式输出

这类现象通常被称为能力涌现，但它不是魔法，背后仍然来自数据、规模、架构和训练目标的共同作用。

## LLM 和传统 NLP 模型的区别

传统 NLP 往往是“一个任务训练一个模型”，比如情感分类、命名实体识别、机器翻译、文本摘要等。LLM 则更像一个通用语言接口。

| 对比维度 | 传统 NLP 模型 | 大语言模型 |
| --- | --- | --- |
| 训练方式 | 通常针对单一任务训练 | 先大规模预训练，再适配多任务 |
| 输入输出 | 固定任务格式 | 自然语言输入，自然语言输出 |
| 泛化能力 | 依赖任务数据 | 可通过 Prompt 迁移到新任务 |
| 使用方式 | 调 API 或模型接口 | 对话、指令、工具调用、Agent |
| 典型能力 | 分类、抽取、翻译等 | 写作、问答、推理、代码、规划 |

这也是为什么 LLM 被称为“基础模型”（Foundation Model）：它不是只服务一个任务，而是可以作为很多应用的底座。

## 主流模型架构路线

LLM 的发展中有几条非常重要的架构路线。

### 1. Encoder-only：理解型模型

代表模型：BERT、RoBERTa

特点：

- 擅长文本理解任务
- 常用于分类、匹配、抽取
- 通过双向上下文建模理解整段文本
- 不适合直接做长文本生成

典型任务：

- 情感分类
- 文本匹配
- 命名实体识别
- 检索向量表示

### 2. Decoder-only：生成型模型

代表模型：GPT、LLaMA、Qwen、DeepSeek

特点：

- 按从左到右的方式生成文本
- 每一步预测下一个 Token
- 非常适合对话、写作、代码和推理
- 是当前主流 LLM 的核心路线

现代大多数聊天大模型都属于 Decoder-only 架构。

### 3. Encoder-Decoder：序列到序列模型

代表模型：T5、BART

特点：

- Encoder 负责理解输入
- Decoder 负责生成输出
- 适合翻译、摘要、问答等任务
- 在大规模通用对话模型时代，热度低于 Decoder-only 路线

## 从预训练模型到 ChatGPT 类模型

一个可用的聊天模型通常不是只靠预训练得到的，而是经历多个阶段。

```text
海量文本预训练
  -> 基础模型 Base Model
  -> 指令微调 SFT
  -> 偏好对齐 RLHF / DPO / GRPO
  -> 对话模型 Chat Model
  -> 面向推理、工具、代码等方向继续增强
```

### 1. 基础模型

基础模型主要通过海量文本预训练得到，具备语言建模能力和大量知识，但不一定擅长听从用户指令。

例如，给基础模型输入：

```text
请解释什么是 Transformer。
```

它可能会继续补全文本，也可能生成不稳定的格式，因为它还没有被充分训练成“助手”。

### 2. 指令模型

指令模型经过 SFT 训练，学习“用户提问 -> 助手回答”的模式。

它更擅长：

- 回答问题
- 按格式输出
- 总结文章
- 写代码
- 执行角色设定

### 3. 对齐模型

对齐模型会进一步学习人类偏好，尽量让回答更有帮助、更安全、更符合用户意图。

常见技术包括：

- RLHF
- DPO
- GRPO
- RLAIF

这些内容会在 [RLHF](../RLHF/README.md) 章节展开。

### 4. 推理模型

推理模型强调复杂问题求解能力，通常会在数学、代码、逻辑、多步规划等任务上做特殊强化。

它们常见特点是：

- 更擅长多步推理
- 能处理复杂数学和代码问题
- 可能使用更长的内部思考或搜索过程
- 推理成本通常更高

## LLM 的关键能力

### 1. 文本生成

LLM 可以生成文章、邮件、总结、故事、脚本、文案和结构化报告。

### 2. 问答与知识整合

LLM 可以基于参数知识或外部检索内容回答问题。结合 RAG 后，可以接入企业知识库和实时文档。

### 3. 翻译与改写

LLM 能够在多语言之间转换，也能进行润色、扩写、压缩和风格迁移。

### 4. 代码生成

LLM 可以补全代码、解释代码、生成测试、定位 Bug，并参与软件工程工作流。

### 5. 推理与规划

通过 Chain-of-Thought、工具调用、搜索和 Agent 框架，LLM 可以参与复杂任务规划。

### 6. 工具调用

LLM 可以不只输出文本，还能决定调用搜索、数据库、计算器、代码解释器、浏览器、业务 API 等工具。

这也是后续 Agent 和 MCP 技术的基础。

## LLM 的局限性

理解 LLM，不能只看能力，也要知道它的边界。

### 1. 幻觉

模型可能生成看似合理但实际错误的内容。它的目标是生成高概率文本，不是天然保证事实正确。

### 2. 知识时效性

模型参数中的知识来自训练数据，无法自动知道训练后发生的新事件。需要联网搜索、RAG 或工具调用补充。

### 3. 长上下文限制

模型能处理的上下文长度有限。即使支持超长上下文，也会面临注意力成本、信息遗忘和定位困难。

### 4. 推理不稳定

LLM 在复杂数学、严格逻辑、长链路规划上仍可能出错，需要评测、工具和校验机制。

### 5. 安全与对齐问题

模型可能被诱导输出不安全内容，也可能受到 Prompt 注入、越狱攻击和数据泄露风险影响。

## 学习建议

如果你刚开始学习 LLM，可以按下面的顺序推进：

1. 先理解“LLM = 根据上下文预测下一个 Token 的大规模神经网络”。
2. 再学习 Tokenizer 和 Embedding，搞清楚文本如何进入模型。
3. 接着学习 Transformer 和 Attention，理解模型如何处理上下文。
4. 然后学习预训练、SFT、RLHF，理解模型能力如何被训练出来。
5. 最后进入 Prompt、RAG、Agent、MCP 和部署等应用层知识。

## 小结

LLM 的表层能力看起来很复杂，但底层主线可以概括为：

```text
文本被切成 Token
  -> Token 被映射为 Embedding
  -> Transformer 处理上下文
  -> 模型预测下一个 Token
  -> 重复预测形成完整输出
  -> 通过 SFT / RLHF / 工具调用变得更像智能助手
```

理解这一点，就抓住了后续学习 LLM 的第一根主线。
